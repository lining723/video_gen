# [PROJECT_NAME] Coding Standards

## Core Conventions

### I. Naming Conventions

Use clear, consistent names that match the surrounding language and repository conventions.

### II. Error Handling

Handle errors in ways that preserve debuggability and keep failure paths explicit.

### III. Logging Standards

Use structured, purposeful logging that helps operators and developers understand system behavior.

## Fixed Rules

- **Style Guides**: Follow the language-specific style guides already used by each service.
- **Formatting Tools**: Use linters and formatters consistently.
- **Atomic Commits**: Keep commits focused and scoped to a single coherent change when possible.

## Governance

- Prefer established patterns in the repository before introducing new abstractions.
- Code review should check correctness, style, and sufficient validation for the change.
- Branch and commit practices should keep `main` deployable and changes easy to review.
