# 智能视频生成平台 Quality

## Core Standards

### I. Quality Gates

所有变更在合并前必须通过构建、审核和验证门禁。

### II. Testing Strategy

测试应覆盖变更的预期行为，并保护关键工作流免受回归影响。

### III. 业务闭环验证

任何变更必须能追溯到业务闭环中的具体价值点，确保不会破坏"创意输入 → 场景设计 → 分镜审核 → 主体出图 → 镜头渲染 → 最终拼接 → 成片交付"的完整链路。

## Fixed Rules

- **CI Required**: 所有服务合并前必须通过 CI。
- **Code Review Required**: 每个变更合并前必须经过审核。
- **Coverage Guardrail**: 测试覆盖率不得在 PR 中下降，除非有明确理由。
- **审核门禁验证**: 场景设计和分镜审核的门禁逻辑必须作为测试覆盖的关键路径。

## Governance

- 质量标准适用于代码、文档和生成的项目资产。
- 当质量门禁失败时，应修复根本问题而不是绕过检查。
- 团队应定期审查质量目标，使其与仓库需求保持一致。
